# Nimlat E2E Coverage Matrix

This matrix is the refactor safety map for Playwright Electron tests. E2E means renderer -> preload/IPC -> main ->
SQLite, with external providers replaced by deterministic doubles.

## Harness Guardrails

| Guardrail                        | Coverage                                                        | Notes                                                                                                                                                                                 |
|----------------------------------|-----------------------------------------------------------------|---------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------|
| Renderer errors fail tests       | `watchRendererConsole`, `assertNoRendererDiagnostics`           | Unknown `console.error`, `pageerror`, and renderer crashes are collected and asserted after every test. Known Chromium/React/Ant noise is whitelisted only in the harness.            |
| Main-process failures fail tests | `getMainProcessDiagnostics`, `assertNoMainProcessDiagnostics`   | Captures unhandled rejections, uncaught exception monitor events, and renderer-process-gone diagnostics from the Electron main process.                                               |
| Isolated app data                | `launchNimlatElectronSession`, `app sandbox survives the suite` | Each suite starts from a temp `appData/userData/sessionData`; cleanup removes the temp root.                                                                                          |
| Real restart persistence         | `user config persistence across real Electron restart`          | Relaunches Electron over the same temp root in `reuse` boot mode, without reseeding fixtures.                                                                                         |
| Typed renderer calls             | `tools/e2e/e2e-renderer-api.ts`                                 | New tests should use `callRendererApi` and renderer event recorders instead of raw string evaluation. Existing raw snippets remain where DOM/Pixi harnesses need custom browser code. |

## Refactor Rules

- Add E2E only when behavior crosses renderer/preload/main/SQLite or protects a user-visible workflow.
- Prefer typed `callRendererApi` for preload calls; use `evaluateRenderer` only for DOM/Pixi interactions or
  browser-only orchestration.
- Every mutation that should update open renderer state should assert either a list invalidation event with affected IDs
  or an item patch event with item IDs.
- Negative tests should assert explicit failure results without calling real external APIs.
- If a test requires restart persistence, use the reusable temp root instead of simulating by closing and reopening the
  database.
